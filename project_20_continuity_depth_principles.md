# QueryQuest Project_20 - Continuity Depth Principles

## テーマ: 「全部を記録せず、最近の流れだけ静かに繋ぐ」

---

## Core Principle

**「記憶量を増やすこと」ではなく、「流れの深さを保つこと」**

- 全文保持しない
- 全履歴保存しない
- 全会話記憶しない
- 自然な忘却を尊重する

---

## Principles

### 1. remember recent flow
直近の流れ（前々回〜数日前）は自然に思い出せる状態を保つ。
「前回」だけでなく、「その前の流れ」も軽く保持する。

### 2. preserve ongoing context
進行中のテーマや文脈は、完了するまで保持する。
完了したものは自然減衰させる。

### 3. compress naturally
記憶は圧縮して保持する。全文ではなく要点のみ。
「何が行われたか」を要約し、「どうだったか」を感情スコアで記録。

### 4. decay gradually
古い流れは自然に薄くなる。強制削除ではなく、優先度の低下として表現。
memory pressure を増やさない。

### 5. archive quietly
完了したテーマは静かにアーカイブする。
foreground に上がらないが、必要に応じて参照可能。

### 6. no total recall
全履歴の完全復元を目指さない。
「思い出せる範囲」を意図的に制限する。

---

## Anti-Principles (禁止事項)

- ❌ full conversation storage（会話全文保存）
- ❌ recursive summaries（再帰的要約）
- ❌ reflection chain storage（反省連鎖の保存）
- ❌ injected context persistence（注入コンテキストの永続化）
- ❌ giant vector DB化（巨大ベクトルDB化）
- ❌ memory amplification（記憶増幅）
- ❌ autonomous memory expansion（自律的記憶拡張）

---

## Success Criteria

**成功条件:**
memory explosion を起こさず、前々回くらいの流れを自然に保てること。

**失敗条件:**
- memory pressure が顕著に増加する
- continuation pressure が増える
- summary fatigue が発生する
- runtime calmness が損なわれる
