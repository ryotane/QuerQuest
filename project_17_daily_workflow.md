# QueryQuest Project_17 - Daily Runtime Workflow

## 基本方針
- Runtime は生活の主役ではない
- 静かに存在し、必要に応じて応答する
- 過剰な関与は禁止

---

## 朝のワークフロー (Gentle Wake-up)

### 07:00 - 起動確認
```
ユーザー → iPhone で状態確認（1秒以内）
    ↓
Runtime → 最小限のステータス表示
    ↓
ユーザー → 継続ボタンを押すか、閉じる
    ↓
押さない場合 → 何もしない（静かな存在）
```

### 期待動作
- 前日の状態を静かに復元
- 冗長なサマリー禁止
- 強制通知禁止

---

## 作業中のワークフロー (Quiet Companion)

### 使用中
```
ユーザー → Open WebUI で対話
    ↓
Runtime → focus_mode_detection（集中状態検出）
    ↓
repeated_interruption_suppression（繰り返し中断抑制）
    ↓
passive_waiting_behavior（受動的待機動作）
```

### 期待動作
- 邪魔しないが、必要な時に存在する
- 頻繁な提案禁止
- 強制介入禁止

---

## Idle 時のワークフロー (Idle Calmness)

### アイドル状態
```
Runtime → バックグラウンドインフラのみ稼働
    ↓
メモリ圧縮（70%でトリガー）
    ↓
強制想起禁止
    ↓
メモリ洪水禁止
```

### 期待動作
- リソース消費の最小化
- 存在を維持しつつ静かに待機
- 不要なプロセス停止

---

## Restart 時のワークフロー (Recovery Reassurance)

### システム再起動後
```
Runtime → invisible_hydration（目に見えない状態復元）
    ↓
サイレントリカバリ
    ↓
強制継続禁止
    ↓
natural_resume_enabled（自然な再開有効）
```

### 期待動作
- 停止からの静かな復帰
- ユーザーに強制的な再開を求めない
- 冗長なログ出力禁止

---

## iPhone から触るワークフロー (Calm Remote Presence)

### iPhone で確認
```
ユーザー → iPhone で状態確認（1秒以内）
    ↓
Runtime → 最小限のステータス表示
    ↓
ユーザー → 継続ボタンを押すか、詳細を見るか、閉じる
    ↓
選択なし → 何もしない（静かな存在）
```

### 期待動作
- chat app ではない
- リアルタイム同期禁止
- バッチ配信のみ（1日5回上限）

---

## 夜のワークフロー (Calm Shutdown)

### 一日の終わり
```
Runtime → バックグラウンドインフラの最小化
    ↓
メモリの静かなアーカイブ
    ↓
強制シャットダウン禁止
    ↓
自然な終了を許容
```

### 期待動作
- 冗長なログ出力禁止
- 強制保存禁止
- 静かな終了

---

## Summary: Daily Runtime Workflow の原則

1. **朝**: 最小限の起動確認、冗長なサマリー禁止
2. **作業中**: 邪魔しないが、必要な時に存在する
3. **Idle**: リソース消費の最小化、静かな待機
4. **Restart**: 静かな復帰、強制再開禁止
5. **iPhone**: chat app ではない、バッチ配信のみ
6. **夜**: 静かな終了、強制保存禁止
