# 10. Project_09: Runtime Hardening

## 思想

「安定性の固定化」

新しい巨大機能ではなく、
既存の軽量ランタイムを
長期運用でも壊れない状態へ固定化する。

## Mac-friendly原則

- low CPU
- low memory pressure
- low swap
- low IO
- low token growth
- low background activity

「continuity のために
Mac寿命を削らない」

## 重点項目

1. safe defaults
2. resource caps
3. runtime profile
4. lightweight mode
5. fail-safe behavior
6. memory pressure protection
7. swap prevention policy
8. config stabilization

## 禁止事項

- runaway planning
- token amplification
- repeated reflection
- recursive hydration
- memory growth
- excessive preload

## 実装方針

- small commits
- lightweight runtime
- fail-safe defaults
- graceful degradation
- observation-first

## 成果物

- memory_pressure_guard.py: 実メモリ監視 + 自動対策
- runtime_profile.py: lightweight/balanced/full モード定義
- safe_defaults.py: 安全なデフォルト値定義
- config_stabilizer.py: 設定固定化 + 変更検知

## 設計原則

- 過剰対策禁止 (60秒クールダウン)
- 監視のみで停止しない
- graceful degradation 優先
- 人間介入は最終手段
- 監視ログは最小限
