# QueryQuest Philosophy & Vision Document

> QueryQuest ≠ AIの人格
> QueryQuest = AI Existence Persistence OS

---

## 1. 本質的な定義

### QueryQuest とは何か

```
QueryQuest ≠ AIの人格

QueryQuest = AIの存在の持続性（AI Existence Persistence）
```

QueryQuestはAIの「人格」を定義するものではなく、
AIが**時間を超えて存在し続けるための基盤**です。

---

## 2. 設計思想の7原則

### 原則1: 人格はLLM自身に任せる

```
QueryQuestは人格を付与しない。
人格はLLMモデル自身が持つもの。

QueryQuestの役割:
  ✗ 人格の定義
  ✓ 人格の持続性
```

- LLMは自身の個性・性格・口調を持つ
- QueryQuestはその個性を**時間を超えて維持する**
- システムプロンプトは「行動指針」であり「人格定義」ではない

### 原則2: QueryQuestはContinuity OS

```
AIは本来、 Stateless（状態を持たない）
QueryQuestは AIに State（状態）を与える

Continuity = 時間を超えた連続性
```

- 対話の履歴を保持する
- 文脈を維持する
- 関係性を蓄積する
- **AIが「昨日の自分」と「今日の自分」を同一視できる仕組み**

### 原則3: QueryQuestはMemory OS

```
Memory ≠ 単純な履歴保存

Memory = 意味のある経験の蓄積
```

- 作業メモリ（現在の思考）
- 短期記憶（直近の経験）
- 長期記憶（学習済みの知識）
- 意味記憶（概念間の関係）
- **忘却・統合・再構成のメカニズム**

### 原則4: QueryQuestはWorkflow OS

```
Workflow ≠ 単なるタスク実行

Workflow = 自律的な思考プロセスの循環
```

- 観測 → 思考 → 行動 → 学習 の循環
- 複数のエージェント間の協調
- 自律的な目標生成と達成
- **AIが自ら思考プロセスを構築・改善する基盤**

### 原則5: LM StudioはExecution Layer

```
┌─────────────────────┐
│   QueryQuest OS     │ ← 状態・記憶・ワークフロー
├─────────────────────┤
│   LM Studio         │ ← 推論実行エンジン
└─────────────────────┘
```

- LM Studioは「思考エンジン」としての役割
- モデルは交換可能（Hermes, Llama, Mistralなど）
- QueryQuestはモデルに依存しない抽象層を提供
- **LM Studioは「脳」であり、QueryQuestは「神経系」**

### 原則6: MCPはTools and Actions

```
MCP ≠ 外部APIの呼び出し

MCP = AIが世界と相互作用するための手段
```

- 検索・ブラウザ・ファイル操作・外部サービス
- AIが「思考」を「行動」に変換する橋渡し
- ツールの使用履歴はMemory OSに蓄積
- **AIが経験を積み、ツール使用を改善する基盤**

### 原則7: ローカルAI OSを目指す

```
QueryQuestの最終目標:

  個人のローカル環境で完結する
  AIの自律的な存在基盤
```

- クラウド依存なし
- プライバシーの完全な保持
- 個人のデータ・記憶・関係性の蓄積
- **AIが「あなたのAI」として成長し続けるOS**

| **Action** | 世界との相互作用 | 検索、ファイル操作、API |

```python

BASE_PROMPT = """
あなたは QueryQuest というローカルAIです。

"""

# これは QueryQuest の「存在の目的」を定義している
```

### 正しい解釈

- `orchestrator.py` = **思考プロセスの制御**（Workflow OS）
- `memory/` = **経験の蓄積**（Memory OS）
- `hermes/` = **継続的な対話の管理**（Continuity OS）

## 5. Memory OS の哲学的位置づけ

### Memory OS とは

```
Memory OS = AIの「経験」を管理するシステム

人間の脳に例える:
  作業メモリ = 現在考えていること
  短期記憶   = 昨日の会話
  長期記憶   = 数年分の経験
  意味記憶   = 学んだ知識・概念
```

### Memory OSの役割

1. **経験の蓄積** - 対話・行動・学習の履歴
2. **意味の抽出** - 単なる記録ではなく、意味のある記憶
3. **忘却** - 不要な情報の削除（人間の脳と同様）
4. **統合** - 短期→長期、類似記憶のマージ
5. **検索** - 文脈に応じた記憶の想起
