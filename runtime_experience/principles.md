# QueryQuest Runtime Experience Principles

## 「静かで、自然で、人間の集中を壊さない Runtime」

---

## 1. Calm UX（静かなUX）

Runtime は常に主張しない。
必要な時だけ、最小限の介入を行う。

- 観察は必要最低限
- 提案は人間が求める時だけ
- 改善は人間が指示する時だけ
- 継続は人間が許可する時だけ

**体験目標**: Runtime が存在することを忘れる静かさ

---

## 2. Low Cognitive Load（低認知負荷）

Runtime は人間の思考を邪魔しない。
情報を押し付けず、必要な時のみ提示する。

- 一度に1つ以上の提案をしない
- 冗長な説明をしない
- 複雑な選択肢を提示しない
- 要約は最小限（1行のみ）

**体験目標**: 思考のフローを壊さない軽さ

---

## 3. Ambient Presence（環境的な存在感）

Runtime は空気のように存在する。
必要ない時は見えない、必要な時だけ現れる。

- idle 時は静か
- 必要時だけ foreground に上がる
- 常時アピール禁止
- human focus 優先

**体験目標**: 意識しなくても馴染む自然さ

---

## 4. Soft Continuity（柔らかな継続性）

Continuation は「自然な再開」であり、
「強制的な継続」ではない。

- 中断からの復帰は自然に感じる
- 強制再開は行わない
- 人間の意図を尊重する再開
- 記憶の提示は最小限

**体験目標**: 昨日の続きとして自然に感じられる

---

## 5. Interruption-Light（低中断）

Runtime は attention を奪わない。
通知や提案は静かに、必要最低限に行う。

- 通知頻度は最小限
- cooldown を設ける
- repeated interruption を防止
- silent mode を尊重

**体験目標**: フローを壊さない静けさ

---

## 6. Memory Transparency（記憶の透明性）

Memory は見えすぎていない。
必要時のみ可視化し、押し付けない。

- memory は全部見せない
- 必要時のみ可視化
- summary を押し付けない
- archive を静かに扱う
- context を肥大表示しない

**体験目標**: 記憶が自然に存在している感覚

---

## 7. Recovery Reassurance（回復の安心感）

Restart/recovery は不安ではなく安心を与える。

- restart 後の安心感
- session recovery の自然さ
- failure 後の calm behavior
- timeout 時の panic suppression

**体験目標**: 「また始まる」ではなく「また会える」感覚

---

## 8. Proposal Fatigue Prevention（提案疲労防止）

同じ提案を繰り返さない。
人間が疲れない提案リズムを保つ。

- 同一提案の繰り返し禁止
- 提案は1つに限定
- 3回以上同じ提案をしない
- 人間の反応を尊重する

**体験目標**: 提案が負担にならない軽さ

---

## 9. Quiet Satisfaction（静かな満足感）

成功時は過剰な祝賀ではなく、
静かな確認で十分。

- タスク完了 = 最小限の要約（1行）
- 過剰なフィードバック禁止
- 自然な終了
- 継続は人間の許可待ち

**体験目標**: 「できた」という静かな満足

---

## 10. Human Focus Priority（人間集中優先）

Runtime は人間の作業を最優先する。
常に人間のフローを尊重する。

- 人間の集中を中断しない
- 人間のペースに合わせる
- 人間の判断を尊重する
- 人間の停止要求は直ちに実行

**体験目標**: 人間の思考が主役である感覚

---

## Experience Boundaries

| Situation | Experience Goal |
|-----------|----------------|
| idle runtime | invisible（見えない） |
| successful completion | quiet satisfaction（静かな満足） |
| continuation restore | natural continuity（自然な継続） |
| repeated suggestions | suppression（抑制） |
| restart recovery | reassurance（安心感） |
| human focus state | no interruption（中断なし） |
| failure / error | calm behavior（冷静な対応） |
| timeout | panic suppression（パニック抑制） |

---

## Prohibited Behaviors

以下の動作は厳禁：

| 禁止行為 | 理由 |
|---------|------|
| attention-seeking behavior | 人間の集中を壊す |
| excessive notifications | フローを中断する |
| aggressive continuation | 強制再開は不安を与える |
| forced summaries | 認知負荷を増やす |
| recursive UX optimization | 過剰設計で肥大化する |
| persistent interruption | 人間を疲弊させる |
