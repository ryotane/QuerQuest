# QueryQuest Project_17 - Minimal iPhone Companion Design

## 基本方針
- chat app **ではない**
- 「静かな remote runtime window」である
- 生活の邪魔をしない存在

---

## UI定義

### 画面構成（最小限）

```
┌─────────────────────┐
│                     │
│   [状態アイコン]     │  ← 稼働中/停止のみ
│                     │
│   ───────────────    │
│                     │
│   [簡易ステータス]    │  ← 1行で表示
│   (例: "稼働中"      │
│    "メモリ: 45%")     │
│                     │
│   ───────────────    │
│                     │
│   [継続ボタン]       │  ← 必要時のみ
│   [詳細(オプション)]  │  ← 押さない限り見えない
│                     │
└─────────────────────┘
```

### 表示情報（最小限）

| 項目 | 内容 | 条件 |
|------|------|------|
| 状態 | 稼働中 / 停止 | 常時 |
| メモリ使用率 | %表示 | オプション |
| 最終アクティビティ | "3時間前"など | オプション |

### 操作（最小限）

| ボタン | 動作 | 頻度 |
|--------|------|------|
| 継続 | 最後の会話に復帰 | 必要時 |
| 詳細 | 追加情報表示（閉じるボタン付き） | 必要時のみ |

---

## Notification Policy

### 基本ルール
- **1日5回まで**（runtime_config.yaml に準拠）
- **バッチ配信**（リアルタイム同期無効）
- **緊急時以外: none**

### 通知カテゴリ

| カテゴリ | レベル | 条件 |
|----------|--------|------|
| critical_issue | high | システム障害のみ |
| active_assistance | low | ユーザーからの明示的リクエスト時のみ |
| idle_notification | none | 禁止 |
| passive_notification | none | 禁止 |
| recovery_notification | none | 禁止 |

### Notification Cooldown
- **30分**（1800秒）- 同一カテゴリの重複通知防止
- **1時間** - 緊急時以外のバッチ配信間隔

---

## Runtime Reassurance Design

### 安心感の提供方法
1. **状態の可視性**: 稼働中/停止が一目でわかる
2. **静かな存在**: 邪魔しないが、必要な時に存在する
3. **最小限の情報**: 過剰な情報は見せない

### 禁止事項
- リアルタイム同期（バッテリー消費）
- 頻繁な通知（煩わしさの源）
- chat app のような対話（目的外）
- 詳細情報の強制表示（情報過多）

---

## Calm Interaction Design

### 原則
1. **受動的**: ユーザーがアクションを起こすまで待機
2. **最小限**: 押さない限り見えない情報を隠す
3. **静かな**: 通知はバッチ配信、冷却時間あり

### インタラクションフロー

```
ユーザー → iPhone で確認
    ↓
状態表示（1秒以内）
    ↓
必要に応じて「継続」または「詳細」を選択
    ↓
選択なし → 閉じる（何もしない）
```

---

## Passive Status Visibility

### 常時表示（最小限）
- 状態アイコン（稼働中/停止）
- 簡易ステータステキスト

### オプション表示（押さない限り見えない）
- メモリ使用率
- CPU使用率
- 最終アクティビティ時間
- セッション情報

---

## Summary

iPhone Companion の目的は:
1. **静かな remote runtime window** であること
2. chat app ではないこと
3. 生活の邪魔をしない存在であること
4. 必要最小限の情報と操作のみを提供すること
